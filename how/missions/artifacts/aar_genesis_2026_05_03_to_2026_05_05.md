## Lessons Learned

1. **Live install catches what no-emacs phases can't.** Three real bugs in the elisp + packages.el surfaced only when a real Spacemacs first-boot tried to load the layer. Recommendation: any future elisp work in `what/standard/layers/adna/` should be byte-compile-validated via `emacs --batch --eval '(byte-compile-file "...")'` BEFORE commit. Add to `skill_health_check` check E.

2. **Plan-driven execution works for self-contained genesis efforts.** A 9-phase plan with stop gates between phases is functionally equivalent to a campaign with phase-missions. The AAR can be free-standing. If the framework convention is "mission file always," that's a candidate for upstream contribution to the .adna template (open question: should the template generate a mission file when forking?).

3. **Sanitization patterns must use `<placeholder>` shape in their own documentation.** Documenting "patterns we forbid" by including literal examples of those patterns is a meta-trap; the scan flags itself. Phase 6 fix established the convention.

4. **The closed loop self-improvement protocol is sound on synthetic friction.** Whether it scales to real friction is empirically TBD; the first real-session ADR will confirm. No ADR cooldown logic was needed in genesis but the spec is in place.

5. **Three-layer architecture (standard/local/overlay) held up unchanged through 8 phases.** The contract did not need amendment during genesis. First amendment likely on first real overlay consumption.

6. **`gh` CLI + repo creation in an org takes ~5 sec; the heavyweight phase of publishing was the dry-run + sanitize step before push, not the push itself.** Future publishes (subsequent path) should be near-instant.

7. **The DoD's "stop on first failure, do not paper over" rule held even when convenient to skip.** Each deferred item (emacs not on host) was documented as a deferral, not papered over as "green." When emacs became available in Phase 8, every deferral was cleared.

8. **The dotfile template's `vterm-always-compile-module t` setting is the kind of subtle batch-mode fix that's invisible until install-from-tarball is exercised.** Reinforces #1 above: ref-clean install validation is essential, not optional.
