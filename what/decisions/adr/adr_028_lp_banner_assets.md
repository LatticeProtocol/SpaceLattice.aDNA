# ADR-028: LP Banner Assets — SVG source + ASCII variants + vault-side storage

## Status

Accepted

## Context

P4-05 calls for LP visual identity assets (startup banner image + ASCII text variants). The original mission spec (written under the two-repo fork model) referenced `core/banners/` paths inside the Spacemacs fork and an `adr_016` placeholder number. Both are superseded:

- The fork model was abandoned in favour of vault-only storage (ADR-024, 2026-05-08). Assets live in `what/standard/layers/spacemacs-latticeprotocol/banners/` and are deployed by `skill_install`.
- ADR-016 was already issued (GC threshold + LSP buffer, 2026-05-07). This decision is correctly numbered ADR-028.

Three asset questions needed decisions:

1. **Image format**: Which formats to commit vs. derive?
2. **ASCII coverage**: How many variants, what constraint?
3. **License**: What posture for wholly original assets?

## Decision

**Image asset**: Commit the SVG source to the vault. PNG (256×256) and ICNS (macOS) are derived at install time via `rsvg-convert` and `sips`/`iconutil`. Only the SVG is a vault artifact; derived formats are build outputs and are gitignored from the layer directory.

**ASCII banners**: Four variants, each ≤75 columns wide, covering four design registers: lattice-grid frame, minimal mark, full box, wings motif. The operator selects via `dotspacemacs-startup-banner` in `what/local/operator.private.el`; the distribution default in `config.el` points to `lp-banner-1.txt` (safe for all terminals).

**License**: GPL-3.0 for all LP-original assets, matching the Spacemacs upstream license. No Spacemacs logo assets (CC BY-SA 4.0) are reused or adapted.

**Storage**: `what/standard/layers/spacemacs-latticeprotocol/banners/`. `skill_install` Step 5 symlinks the entire layer directory; no additional deploy step is required for text assets. PNG/ICNS derivation is a one-time operator step documented in `spacelattice_assets.md`.

## Consequences

### Positive

- SVG-first approach: the vault holds a lossless, editable source; operators can regenerate PNG at any size.
- Four ASCII variants give operators variety without requiring image support.
- GPL-3.0 posture keeps the distribution license clean and coherent.
- Vault-only storage (no fork path) is consistent with ADR-024.

### Negative

- PNG/ICNS are not auto-generated by `skill_install`; operators must run the derivation step manually (one-time, documented).
- No official LP graphic designer review — assets are agent-authored; visual refinement is deferred to post-v1.0.

### Neutral

- The `dotspacemacs-startup-banner` override in `config.el` pins `lp-banner-1.txt` as the distribution default. Operators override in `what/local/operator.private.el` per ADR-024 layered-precedence rules.
