# Obsidian coupling

Spacemacs and Obsidian are first-class peers in the operator's workflow. Spacemacs handles agentic editing; Obsidian handles visual graph navigation. This spec defines how they round-trip.

## Threat model — what we're NOT doing

- **No bidirectional state replication.** Each tool is authoritative for its own state.
- **No Obsidian-side plugin authoring.** This vault is plain markdown that Obsidian opens natively. We rely on Obsidian's existing plugins, not new ones.
- **No CRDT or conflict resolution.** If both tools save the same file at the same time, last-write-wins (Spacemacs and Obsidian both notice external changes via `auto-revert-mode` and `Detect all file changes`).

## Round-trip directions

### A. Spacemacs → Obsidian (active)

Implemented in Phase 4 as `M-x adna/open-in-obsidian`.

Mechanism: build an Obsidian Advanced URI and `browse-url` it. Requires the operator to have the **Advanced URI plugin** installed in Obsidian (a community plugin by `Vinzent03/obsidian-advanced-uri`).

```elisp
(defun adna/open-in-obsidian ()
  (interactive)
  (let* ((root (adna/find-vault-root default-directory))
         (relpath (file-relative-name buffer-file-name root))
         (vault-name (file-name-nondirectory (directory-file-name root)))
         (uri (format "obsidian://adv-uri?vault=%s&filepath=%s"
                      (url-hexify-string vault-name)
                      (url-hexify-string relpath))))
    (browse-url uri)))
```

Bound to `SPC a o`.

### B. Obsidian → Spacemacs (passive, opt-in)

Off by default. Enabled by `(setq adna-obsidian-roundtrip-enabled t)` in `what/local/operator.private.el`.

Mechanism (Phase 4 implementation): file-watcher on `<vault-root>/.obsidian/workspace.json` (this file is gitignored but always present locally — Obsidian rewrites it whenever the operator switches active file). The watcher:

1. Polls every 2 seconds (cheap — small JSON file).
2. Parses the `lastClosedFiles` and `activeLeafId` to determine "what file does Obsidian have focused right now?"
3. If different from the last-known state, asks Spacemacs to `find-file` that path (only if Spacemacs is the foreground app — uses `frame-focus-state`).

Trade-offs of this approach:

- **Pro**: zero plugin authoring; works with vanilla Obsidian.
- **Con**: 2-sec lag; doesn't catch sub-second switches.
- **Con**: polling overhead even when Obsidian isn't running (mitigation: skip if `.obsidian/workspace.json` mtime unchanged for 60s).

### C. Wikilinks (built-in)

Operators write `[[Target]]` in their notes. Both tools resolve them:

- **Spacemacs**: `adna/wikilink-follow` (see `adna-bridge.md` § Wikilink follow)
- **Obsidian**: native wikilink resolution

Spacemacs resolution mimics Obsidian's behavior closely enough that round-trip clicking works. Edge cases (case-sensitivity, ambiguous targets) are documented in `adna-bridge.md`.

## What lives in `.obsidian/`

The `.obsidian/` directory is partially gitignored:

| File | Tracked? | Why |
|------|----------|-----|
| `app.json`, `appearance.json`, `core-plugins.json`, `community-plugins.json`, `hotkeys.json` | ✅ | Shared across operators (the standard layer) |
| `workspace.json`, `workspace-mobile.json` | ❌ | Per-device state (active pane, layout) |
| `graph.json` | ❌ | Per-device graph view position |
| `plugins/`, `themes/` | ❌ | Per-device installs |

The fork stripped per-device files (`plugins/`, `themes/`, `workspace.json`, `graph.json`) at fork time. The remaining tracked files are the shared Obsidian config baseline.

## Plugins recommended

For full round-trip, operators install (locally — these get gitignored):

| Plugin | Purpose |
|--------|---------|
| Advanced URI (Vinzent03) | Spacemacs → Obsidian routing |
| Templater | Frontmatter scaffolding (interoperates with `template_session.md` etc.) |
| Dataview | Render frontmatter-aware queries |
| Graph Analysis | Visualize the context graph generated by `adna-index-project` |

These are operator-local choices — they're listed here as recommendations but not enforced.

## Failure modes

- **Obsidian not running** → `adna/open-in-obsidian` falls back to `find-file` in current Spacemacs frame, with a message to start Obsidian if the operator wants the visual.
- **Advanced URI plugin missing** → first invocation fails silently; on second, prompt to install or disable round-trip.
- **`.obsidian/workspace.json` malformed** → watcher logs to `*Messages*` and stops polling for 5 minutes.
